# Project Scope Statement

## Project Title

TechFix Solutions Website

## Project Purpose

The purpose of this project is to design and develop a professional two-page website for a fictional computer repair business called TechFix Solutions. The website will provide information about the company's services, mission, and contact information while demonstrating project planning, documentation, and basic web development skills learned throughout the course.

## Project Description

This project will consist of planning, designing, developing, and publishing a two-page website using HTML and CSS. The website will include a Home page and an About page connected through navigation links. All planning documents and website files will be organized within a public GitHub repository and deployed using GitHub Pages.

## Desired Results

- Create a clean and professional website.
- Demonstrate effective project planning and organization.
- Publish the website using GitHub Pages.
- Create complete project documentation using Markdown.
- Showcase HTML, CSS, GitHub, and project management skills.

## Project Deliverables

- Public GitHub repository
- README.md
- scope.md
- plan.md
- retrospective.md
- index.html
- about.html
- style.css
- Live GitHub Pages website

## Project Requirements

- Two HTML pages with working navigation
- One external CSS stylesheet
- Professional and responsive layout
- Organized GitHub repository
- Complete project documentation
- Published website through GitHub Pages

## Project Exclusions

This project will not include:

- Online appointment scheduling
- Customer login accounts
- Shopping cart functionality
- Payment processing
- Database integration
- JavaScript functionality
- Mobile application development

## Constraints

- Complete the project before the course deadline.
- Use HTML, CSS, Markdown, GitHub, and GitHub Pages.
- Limit the website to two pages.
- Follow all assignment requirements provided by the instructor.

## Assumptions

- GitHub Pages will successfully host the website.
- Required software will remain available.
- Internet access will be available throughout the project.
- The project can be completed using the skills learned during the course.

## Success Criteria

The project will be considered successful if:

- All required files are included.
- The GitHub repository is public.
- The website is functional and easy to navigate.
- All documentation is complete.
- The website is successfully deployed using GitHub Pages.
